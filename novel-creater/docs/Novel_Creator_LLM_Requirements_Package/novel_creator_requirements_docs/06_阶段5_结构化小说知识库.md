# 06 阶段 5：结构化小说知识库

> 定义角色、世界观、笔记、大纲、时间线和一致性检查所需的数据与交互。

## 1. 阶段目标

阶段 5 让 Novel Creator 从普通 AI 编辑器变成小说创作工作台。知识库承载角色、世界观、时间线、伏笔、素材和大纲，让 Agent 能长期理解项目，而不是只看当前聊天窗口。

## 2. 大纲模型

| 字段 | 说明 |
| --- | --- |
| id/projectId | 唯一标识和项目归属。 |
| parentId/order | 支持卷/章/节/场景嵌套和排序。 |
| title | 节点标题。 |
| nodeType | volume/chapter/scene/beat/custom。 |
| summary | 节点摘要。 |
| linkedChapterId | 绑定正文章节。 |
| status | planned/writing/done/archived。 |

## 3. 角色卡

角色卡建议字段：
```text
Character {
  id, projectId, name, aliases,
  role: protagonist | antagonist | supporting | minor | custom,
  description, appearance, personality,
  goals, conflicts, secrets,
  relationships: [{targetCharacterId, relationType, description}],
  firstAppearanceChapterId?, tags,
  consistencyFacts: [Fact]
}
```

Fact 用于一致性检查，例如年龄、称呼、能力限制、禁忌、重要经历。

## 4. 世界观设定

| 分类 | 示例字段 |
| --- | --- |
| 地理 | 地点、地貌、距离、势力范围。 |
| 历史 | 年代、重大事件、纪年法。 |
| 规则系统 | 魔法、科技、超能力、社会制度。 |
| 组织势力 | 组织目标、成员、资源、敌对关系。 |
| 文化习俗 | 语言、礼仪、宗教、节日、禁忌。 |
| 物品道具 | 来源、能力、限制、持有人。 |

## 5. 笔记与资料卡

Note 分为四类：
- idea：灵感。
- research：调研资料。
- decision：用户确认过的设定决策。
- issue：待修复问题。

资料卡必须区分“来源摘要”和“创作用改写”。联网资料不得直接混入正文，除非用户明确改写并确认。

## 6. 一致性检查类型

| 检查类型 | 输入 | 输出 |
| --- | --- | --- |
| 角色一致性 | 角色卡、章节正文、角色事实 | 疑似矛盾、位置、建议修复 |
| 时间线一致性 | 章节顺序、时间线事件、正文时间表达 | 时间冲突、缺失节点 |
| 世界观规则 | 设定条目、正文事件 | 规则违背、例外是否解释 |
| 伏笔回收 | 笔记/大纲/正文线索 | 未回收伏笔、可能回收章节 |
| 视角人称 | 章节正文、写作偏好 | 人称漂移、视角越界 |
| 风格一致性 | 风格配置、章节样本 | 偏离程度、修改建议 |

## 7. 知识库与 Agent 的关系

Agent 使用知识库时必须注明引用范围：
- 本次使用了哪些角色卡。
- 本次使用了哪些世界观条目。
- 本次使用了哪些笔记或调研资料。
- 哪些内容缺失导致建议可信度下降。

用户应能从 Agent 回复跳转到相关知识条目。

## 8. UX 要求

- 左侧栏提供“大纲 / 角色 / 世界观 / 笔记”切换。
- 编辑器中选中文本可创建角色事实或笔记。
- Agent 生成设定更新时，先进入“建议变更”，用户确认后写入知识库。
- 知识库条目支持标签和全文搜索。
- 角色卡和世界观不应强制用户一开始填写，可从正文中逐步抽取。

## 9. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 创建角色 | 角色卡保存后可被章节和 Agent 引用。 |
| 创建设定 | 世界观条目可分类、搜索、编辑。 |
| 笔记入库 | 用户或搜索摘要可保存为 Note，并保留来源。 |
| 从正文抽取 | 选中文本可生成角色事实建议，确认后写入。 |
| 一致性检查 | 至少能检查角色事实与当前章节的明显矛盾。 |
| 上下文引用 | Agent 回复显示本次使用的知识库条目。 |
