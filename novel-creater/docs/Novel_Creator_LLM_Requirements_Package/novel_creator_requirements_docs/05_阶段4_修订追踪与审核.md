# 05 阶段 4：修订追踪与审核

> 定义 AI 修改如何进入正文、diff 粒度、接受/拒绝、撤销/重做、冲突和修订历史。

## 1. 阶段目标

阶段 4 是产品专业性的核心。Agent 不能直接覆盖作者正文，而是以可审查、可撤销、可追溯的修订进入编辑器。该阶段要把“AI 生成”转化为“作者可控的编辑建议”。

## 2. 修订粒度

| 粒度 | 用途 | 默认策略 |
| --- | --- | --- |
| 整段级 | 续写、扩写、缩写、段落改写 | 默认使用，降低审核疲劳。 |
| 句子级 | 润色、语气调整 | 作为可选精细模式。 |
| 字符/词级 | 拼写、标点、细微措辞 | 不建议默认展示，容易造成噪音。 |

## 3. RevisionPatch 协议

```text
RevisionPatch {
  chapterId,
  baseContentHash,
  operation: insert | delete | replace,
  anchor: {
    type: range | paragraphId | cursor,
    start?, end?, paragraphId?, offset?
  },
  beforeText,
  afterText,
  source: agent | user,
  metadata: { prompt, model, taskId, summary }
}
```

baseContentHash 用于判断 Agent 生成期间正文是否已变化。若 hash 不一致，应进入冲突处理。

## 4. 审核模式

| 模式 | 说明 | 适用场景 |
| --- | --- | --- |
| 单条审核 | 逐条接受/拒绝 Revision | 精修段落、敏感修改 |
| 段落审核 | 一个段落内的多个修改整体处理 | AI 改写整段 |
| 整体审核 | 接受/拒绝一次 AgentTask 的全部修改 | 续写大段或批量润色 |
| 预览模式 | 原文/新文/修订视图切换 | 用户比较整体效果 |

## 5. 接受/拒绝规则

接受修订：
- 将 afterText 应用到正文。
- Revision.status = accepted。
- 记录 resolvedAt。
- 操作进入 undo 栈。

拒绝修订：
- 保留或恢复 beforeText。
- Revision.status = rejected。
- 不改变用户正文。
- 操作进入 undo 栈。

若用户手动编辑了包含 pending revision 的区域：
- 将原 revision 标记为 superseded。
- 提示“该修订已因手动编辑失效”。

## 6. 冲突处理

| 冲突 | 处理 |
| --- | --- |
| 生成期间用户编辑了同一区域 | 不自动应用；显示冲突，提供复制建议/重新生成。 |
| 章节已被锁定或删除 | AgentTask 失败，保留输出为会话消息。 |
| baseContentHash 不一致 | 尝试按 anchor 重新定位；失败则进入手动应用。 |
| 多条修订重叠 | 按创建时间排序；后续修订依赖前一条状态。 |

## 7. Undo/Redo 规则

所有修订相关操作都必须进入编辑器命令栈：

```text
EditorCommand
  - InsertText
  - DeleteText
  - ReplaceRange
  - CreateRevision
  - AcceptRevision
  - RejectRevision
  - AcceptRevisionGroup
  - RejectRevisionGroup
```

撤销“接受修订”时，应恢复 pending 状态或恢复原文，具体取决于实现复杂度；最低要求是恢复正文内容并保留修订历史。

## 8. 修订历史面板

修订历史应展示：
- 时间
- 来源：用户/Agent
- 工具类型：rewrite、expand、polish 等
- 摘要
- 状态：待处理/已接受/已拒绝/已失效
- 操作：定位、查看前后对比、复制建议、回滚到该状态

## 9. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 生成修订 | Agent 改写选区后正文显示 pending revision，不直接覆盖。 |
| 接受修订 | 正文变为 afterText，状态为 accepted，可撤销。 |
| 拒绝修订 | 正文保持 beforeText，状态为 rejected。 |
| 冲突检测 | 生成期间修改原文后，系统不强行套用旧 patch。 |
| 批量审核 | 一次 AgentTask 的多条修订可整体接受/拒绝。 |
| 历史追踪 | 每条 AI 修改可追溯 prompt、model、时间和结果。 |
