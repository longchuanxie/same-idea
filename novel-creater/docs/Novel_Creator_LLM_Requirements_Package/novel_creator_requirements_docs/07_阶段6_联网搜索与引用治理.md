# 07 阶段 6：联网搜索与引用治理

> 定义搜索流程、网页提取、资料卡、引用、缓存、版权边界和前端架构限制。

## 1. 阶段目标

阶段 6 提供可控的联网调研能力。Agent 可以搜索、提取、总结资料，并写入项目资料库，但不能把外部内容无约束地直接变成小说正文。重点是可信来源、可追溯引用、用户确认和版权边界。

## 2. 搜索模式

| 模式 | 说明 | 默认策略 |
| --- | --- | --- |
| 手动搜索 | 用户输入查询词或从 Agent 面板触发 | 默认启用，风险低。 |
| Agent 建议搜索 | Agent 提出需要查哪些问题，用户确认后执行 | 推荐默认。 |
| 自动深度搜索 | Agent 多轮搜索、提取、总结 | 高级模式，需明确开关和预算。 |

## 3. 搜索流程

```text
用户问题/Agent 查询建议
  -> SearchPlan(query, intent, maxResults, language)
  -> SearchProvider.search()
  -> WebExtractor.fetchAndClean()
  -> SourceCard(title, url, snippet, extractedText, credibilityHint)
  -> ResearchNote(summary, quotes?, citations)
  -> 用户确认保存到资料库
```

## 4. SourceCard 字段

```text
SourceCard {
  id, projectId, query,
  title, url, domain, publishedAt?, retrievedAt,
  snippet, extractedText?, summary,
  credibilityHint?, language,
  cacheKey, tags
}
```

retrievedAt 必须记录，因为网页内容会变化。

## 5. 引用规则

| 规则 | 说明 |
| --- | --- |
| 资料引用 | 调研笔记必须保留 URL、标题、检索时间。 |
| 正文隔离 | 小说正文默认不插入引用脚注，除非用户选择非虚构/注释模式。 |
| 短引用限制 | 允许保存短摘录用于参考，但不鼓励长段复制。 |
| 摘要改写 | Agent 应输出消化后的资料摘要，不直接复述网页长文本。 |
| 来源可信度 | 不自动声明权威；只显示来源域名、时间和可疑提示。 |

## 6. 前端架构限制与降级

| 问题 | 风险 | 降级方案 |
| --- | --- | --- |
| Web 端 API Key | 前端直连搜索/LLM 可能暴露密钥 | 提示风险或提供用户自托管代理配置。 |
| CORS | 浏览器无法直接抓取很多网页 | 使用搜索 API 摘要；或让用户粘贴内容；或配置代理。 |
| 反爬/动态页面 | 正文提取失败 | 保存搜索结果摘要，提示“无法提取全文”。 |
| 长任务中断 | 切后台或刷新导致任务停止 | 任务状态持久化，允许重试。 |
| 搜索成本 | 深度搜索可能费用高 | 设置最大结果、最大页面、预算提示。 |

## 7. 版权与安全提示

产品应内置提示：
- 调研资料仅用于理解背景，不应大段复制到正文。
- 对真实人物、医学、法律、金融、历史事实等内容，Agent 结果仅供创作参考，需要用户自行核实。
- 若搜索涉及敏感、成人、暴力或受限主题，应遵守服务商和平台政策，并提示用户风险。

## 8. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 手动搜索 | 输入查询词后返回结果列表，包含标题、URL、摘要。 |
| 保存资料 | 搜索结果可保存为 Research Note，并保留 retrievedAt。 |
| 提取失败 | CORS/抓取失败时有提示，不阻断搜索结果使用。 |
| Agent 调研 | Agent 能生成搜索计划，用户确认后执行。 |
| 引用展示 | 资料卡中可查看来源 URL、查询词、检索时间。 |
| 缓存命中 | 重复搜索同一 URL 时优先使用缓存并提示更新时间。 |
