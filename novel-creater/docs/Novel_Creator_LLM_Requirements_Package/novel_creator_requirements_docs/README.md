# Novel Creator — LLM 驱动的小说创作 Agent 需求设计文档包

本包用于 LLM 驱动实施。阶段代表工程依赖层与验收切片，不代表传统产品版本。

## 文件清单
- `00_需求总纲与实施原则.md`：定义产品目标、用户价值、LLM 实施方式、全局边界和验收方法。
- `01_阶段0_领域模型与状态协议.md`：统一实体、状态机、权限、副作用和跨模块契约，是后续 LLM 实施的地基。
- `02_阶段1_基础工程与本地存储.md`：定义 Flutter 工程骨架、模块边界、本地数据库、设置、安全和迁移策略。
- `03_阶段2_项目工作台与编辑闭环.md`：定义项目空间、章节树、编辑器基础行为、保存、恢复和最基础导出闭环。
- `04_阶段3_Agent基础能力与工具契约.md`：定义 LLM Client、对话、工具调用、AgentTask 状态、写作类工具输入输出。
- `05_阶段4_修订追踪与审核.md`：定义 AI 修改如何进入正文、diff 粒度、接受/拒绝、撤销/重做、冲突和修订历史。
- `06_阶段5_结构化小说知识库.md`：定义角色、世界观、笔记、大纲、时间线和一致性检查所需的数据与交互。
- `07_阶段6_联网搜索与引用治理.md`：定义搜索流程、网页提取、资料卡、引用、缓存、版权边界和前端架构限制。
- `08_阶段7_高级会话分支与时光机.md`：定义多 Session、上下文摘要、分支、快照、回溯和合并的边界，避免概念失控。
- `09_阶段8_导出与多端体验.md`：定义 TXT、Markdown、EPUB、PDF、DOCX 导出，以及桌面、平板、手机的响应式交互。
- `10_验收测试矩阵.md`：把各阶段需求转成可执行测试清单，供 LLM 逐项生成与验证。
- `11_LLM实施工作流与提示词模板.md`：把需求转成 LLM 可执行的工作流、任务拆分和提示词模板。

## 使用建议

按 00 到 11 阅读。每次让 LLM 实施一个模块时，至少提供 00、相关阶段文档、10 验收测试矩阵、11 提示词模板。
