# 03 阶段 2：项目工作台与编辑闭环

> 定义项目空间、章节树、编辑器基础行为、保存、恢复和最基础导出闭环。

## 1. 阶段目标

阶段 2 建立用户真正能写作的工作台：创建项目、管理章节、打开编辑器、编辑正文、自动保存、恢复内容、导出 Markdown/TXT。该阶段是产品可用性的最小闭环，不依赖高级 Agent。

## 2. 主界面信息架构

| 区域 | 桌面端 | 移动端 |
| --- | --- | --- |
| 项目入口 | 项目列表 + 最近项目 | 项目列表首页 |
| 左侧栏 | 大纲/章节/角色/资料切换 | 抽屉或底部导航进入 |
| 中间区 | 编辑器主区域 | 单页编辑器 |
| 右侧栏 | Agent 面板，可折叠 | 独立 AI 页面或底部 Sheet |
| 顶部栏 | 项目标签、保存状态、导出、设置 | 标题、更多菜单、保存状态 |

## 3. 项目创建流程

用户流程：
1. 点击“新建项目”。
2. 输入标题、作者、语言、类型，可选一句创意。
3. 系统创建项目、默认大纲根节点和第一章。
4. 进入工作台，光标定位到第一章正文。

默认项目结构：
```text
Project
  OutlineRoot
    第一卷（可选）
      第一章
  Chapter: 第一章
  Session: 默认助手
```

## 4. 章节树需求

| 能力 | 要求 |
| --- | --- |
| 新建章节 | 可在当前节点后、子级或末尾创建。 |
| 重命名 | 标题变更应同步到大纲节点和导出目录。 |
| 排序 | 支持拖拽或上移/下移；order 字段稳定。 |
| 状态标识 | 显示 draft/reviewing/revised/published。 |
| 字数显示 | 章节节点显示缓存字数。 |
| 软删除 | 删除前确认；默认进入回收区或可撤销。 |

## 5. 编辑器基础行为

推荐默认正文格式为 Markdown。编辑器应具备：
- 标题、正文、引用、列表、分隔线等基础写作格式。
- 自动保存：停止输入 1-2 秒后保存，切换章节前强制保存。
- 保存状态：已保存、保存中、保存失败、本地冲突。
- 字数统计：章节字数、项目总字数、今日新增字数。
- 撤销/重做：用户编辑必须进入 undo/redo 栈。
- 只读状态：导出中、迁移中、章节锁定时显示只读提示。

## 6. 内容保存策略

保存策略建议：
```text
用户输入 -> EditorState -> debounce -> ChapterRepository.saveContent()
                     -> update plainTextCache / wordCount / updatedAt
```

约束：
- 切换章节前不得丢失未保存内容。
- 保存失败时不能清空编辑器，应保留本地内存态并提示重试。
- 长章节保存应避免全项目重写。

## 7. 基础导出

| 格式 | 阶段 2 要求 |
| --- | --- |
| TXT | 按章节标题和正文顺序拼接，章节间空行分隔。 |
| Markdown | 保留标题层级、正文 Markdown、章节顺序。 |
| 项目包 JSON | 导出完整项目数据，用于备份和迁移。 |

## 8. UX 关键细节

- 用户首次进入工作台，应看到“开始写作”而不是复杂配置。
- Agent 面板可以存在但默认折叠或显示轻量入口。
- 多项目标签页仅在桌面端展示；移动端回到项目列表切换。
- 保存状态必须始终可见，降低创作焦虑。

## 9. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 新建项目 | 创建后自动进入第一章，关闭重开仍可打开。 |
| 编辑保存 | 输入正文后自动保存，重启后内容一致。 |
| 章节管理 | 新建、重命名、排序、删除后结构正确。 |
| 导出 TXT/MD | 导出文件包含全部章节且顺序正确。 |
| 保存失败 | 模拟写入失败时，用户内容不丢失并可重试。 |
