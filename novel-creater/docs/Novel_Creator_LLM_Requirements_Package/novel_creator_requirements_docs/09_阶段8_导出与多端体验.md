# 09 阶段 8：导出与多端体验

> 定义 TXT、Markdown、EPUB、PDF、DOCX 导出，以及桌面、平板、手机的响应式交互。

## 1. 阶段目标

阶段 8 完善成稿交付和多端体验。导出不是简单保存文本，而是将项目结构、章节顺序、标题层级、元数据和用户排版配置转成可交付文件。多端体验则要避免把桌面三栏布局硬塞进手机。

## 2. 导出流程

```text
选择导出格式
  -> 选择范围：全书/卷/章节/当前章
  -> 选择模板与排版设置
  -> 预览
  -> 生成文件
  -> 记录 ExportLog
```

导出不应修改项目正文，但可以记录导出日志。

## 3. 格式要求

| 格式 | 要求 |
| --- | --- |
| TXT | 章节标题 + 纯文本正文；可选空行、分隔线、编码 UTF-8。 |
| Markdown | 保留标题层级、列表、引用、注释；适合二次编辑。 |
| EPUB | 生成目录、元数据、封面、章节导航；正文转 XHTML。 |
| PDF | 页面尺寸、边距、字体、字号、页码、目录；适合打印和投稿预览。 |
| DOCX | Word 兼容标题样式、正文样式、目录占位、基本格式保留。 |
| Project Package | 完整项目备份，包含数据、附件、导出日志和 schemaVersion。 |

## 4. 导出设置

| 设置 | 说明 |
| --- | --- |
| 章节编号 | 使用原标题/自动编号/隐藏编号。 |
| 作者信息 | 可写入元数据或封面。 |
| 封面 | EPUB/PDF 可选。 |
| 字体字号 | PDF/DOCX 可配置；缺失字体需降级。 |
| 是否包含注释 | 默认不包含内部笔记；用户可选择附录。 |
| 是否包含修订 | 默认导出已接受正文；pending revision 需提示。 |

## 5. Pending Revision 导出规则

当存在待处理修订时，导出前必须提示：
- 仅导出当前正文，不包含未接受修订。
- 导出包含修订标记版本。
- 先处理所有修订。

默认选项应为“仅导出已接受内容”，避免把未确认 AI 文本混入成稿。

## 6. 响应式布局

| 设备 | 推荐交互 |
| --- | --- |
| 桌面 | 三栏布局：左结构/中编辑/右 Agent；支持项目标签页。 |
| 平板横屏 | 三栏可折叠，Agent 可浮层。 |
| 平板竖屏 | 左栏抽屉，中间编辑，Agent 作为侧滑面板。 |
| 手机 | 底部导航：正文、大纲、资料、AI；单任务页面优先。 |
| Web | 注意 API Key 暴露、浏览器下载、CORS 限制。 |

## 7. 快捷键

| 操作 | 桌面快捷键建议 |
| --- | --- |
| 打开 Agent | Cmd/Ctrl + J |
| 续写 | Cmd/Ctrl + Enter |
| 接受修订 | Cmd/Ctrl + Shift + A |
| 拒绝修订 | Cmd/Ctrl + Shift + R |
| 保存 | Cmd/Ctrl + S |
| 全局搜索 | Cmd/Ctrl + K |
| 专注模式 | Cmd/Ctrl + Shift + F |

## 8. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 导出 TXT/MD | 全书章节顺序正确，特殊字符不乱码。 |
| 导出 DOCX | 标题和正文样式可在 Word/LibreOffice 打开。 |
| 导出 PDF | 页码、标题、正文无明显错位。 |
| 导出 EPUB | 可被常见阅读器打开，目录可跳转。 |
| 移动端写作 | 手机可完成打开章节、编辑、保存、询问 AI。 |
| Pending 提示 | 存在未处理修订时导出前出现明确选择。 |
