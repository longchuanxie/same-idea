# 08 阶段 7：高级会话、分支与时光机

> 定义多 Session、上下文摘要、分支、快照、回溯和合并的边界，避免概念失控。

## 1. 阶段目标

阶段 7 引入高级创作探索能力：多个对话方向、上下文摘要、项目快照、分支尝试和时光机回溯。该阶段必须强调“可理解”和“可恢复”，避免用户不知道自己处在哪条时间线。

## 2. Session 模型

```text
Session {
  id, projectId, title,
  stage: brainstorm | research | outline | writing | polish | custom,
  parentSessionId?, branchName?,
  messages,
  contextSnapshotId?,
  createdAt, updatedAt, archived
}
```

Session 是对话容器，不等同于项目版本。对话分支不应自动改变项目正文，除非用户确认应用结果。

## 3. 上下文摘要

当消息超过上下文窗口时，系统生成 ContextSummary：
```text
ContextSummary {
  id, sessionId, coveredMessageRange,
  summary,
  keyDecisions,
  unresolvedQuestions,
  referencedEntities,
  createdAt,
  model
}
```

摘要不能替代原始消息。用户应能展开查看原始对话。

## 4. 快照与时光机边界

| 概念 | 说明 |
| --- | --- |
| Snapshot | 项目数据在某一时间点的只读备份。 |
| Manual Snapshot | 用户手动创建，适合重大修改前。 |
| Auto Snapshot | 迁移、批量修订、导入前自动创建。 |
| Time Machine | 从快照恢复或基于快照创建分支。 |
| Branch | 基于某快照复制出的项目状态，用于探索不同剧情方向。 |

## 5. 回溯规则

必须避免“对话回溯”和“项目回溯”混淆：
- 回到某条对话消息：只影响当前 Session 后续对话，不自动回滚正文。
- 恢复项目快照：会改变项目数据，必须二次确认，并先创建当前状态备份。
- 基于快照创建分支：推荐默认方式，安全性高于直接覆盖当前项目。

## 6. 分支合并

| 能力 | 要求 |
| --- | --- |
| 分支命名 | 创建时必须命名，例如“黑暗结局路线”。 |
| 差异查看 | 展示章节、大纲、角色、设定的差异摘要。 |
| 选择性合并 | 允许只合并某章节或某角色卡。 |
| 冲突提示 | 同一实体两边均修改时，用户选择保留哪一版。 |
| 不可自动合并正文 | 长文本冲突必须由用户确认。 |

## 7. UX 要求

- 顶部明确显示当前项目/当前分支。
- 时光机入口不应放在高频写作按钮附近，避免误触。
- 恢复快照前显示影响范围：章节数、角色数、笔记数、会话数。
- 默认推荐“创建分支继续”，而不是“覆盖当前项目”。

## 8. 阶段验收

| 场景 | 通过标准 |
| --- | --- |
| 多会话 | 同一项目可创建多个 Session，消息互不混淆。 |
| 摘要生成 | 长对话生成摘要后，后续 Agent 可引用摘要。 |
| 手动快照 | 用户可创建快照并查看创建时间和说明。 |
| 基于快照分支 | 可从快照创建分支项目/分支状态。 |
| 恢复确认 | 恢复快照前有影响范围和当前备份提示。 |
| 差异查看 | 分支与主线至少能展示章节标题和正文变更摘要。 |
