# 02 阶段 1：基础工程与本地存储

> 定义 Flutter 工程骨架、模块边界、本地数据库、设置、安全和迁移策略。

## 1. 阶段目标

阶段 1 负责建立可持续扩展的工程基础。目标不是做漂亮界面，而是确保项目结构、依赖注入、存储仓库、设置管理和错误处理足够稳定，后续 LLM 生成模块时不会互相污染。

## 2. 推荐目录结构

```text
lib/
  app/                 # 路由、主题、应用壳
  core/                # 通用错误、结果类型、日志、时间、ID
  domain/              # 实体、枚举、领域服务、接口
  data/                # 本地存储、DTO、Repository 实现
  agent/               # Agent Engine、Tool Registry、LLM Client
  editor/              # 编辑器模型、命令、diff、revision
  features/
    projects/
    workspace/
    knowledge_base/
    settings/
    export/
  infra/
    llm/
    search/
    file_system/
```

## 3. 架构原则

| 原则 | 说明 |
| --- | --- |
| Domain 不依赖 UI | 实体和业务规则不引用 Flutter Widget。 |
| Agent 不直接写 UI | Agent 返回结构化 Command / Patch，由应用层应用。 |
| Repository 封装存储 | UI 不直接访问 Hive/Isar Box。 |
| 错误显式化 | 使用 AppResult 或类似结构表达成功/失败。 |
| 可替换基础设施 | LLM、Search、Export、Storage 都通过接口隔离。 |

## 4. 本地存储选型

推荐使用 Isar 或 Hive 均可。若强调查询、索引和关系，可优先 Isar；若强调简单 Key-Value 和快速落地，可优先 Hive。无论使用哪种，都应通过 Repository 层隔离，避免上层绑定具体数据库。

建议存储分区：
- projects：项目元数据。
- chapters：章节正文与缓存字段。
- revisions：修订记录。
- knowledge：角色、世界观、笔记、资料卡。
- sessions：对话、Agent 任务、上下文快照。
- settings：模型、服务商、主题、写作偏好。
- export_logs：导出记录。

## 5. 设置与密钥管理

| 设置项 | 要求 |
| --- | --- |
| 服务商配置 | base_url、api_key、model、display_name、enabled。 |
| API Key | 本地加密保存；Web 端必须提示暴露风险或使用代理。 |
| 默认模型 | 全局默认 + 项目覆盖 + 单次对话临时切换。 |
| 写作偏好 | 语言、默认风格、是否自动建议、默认生成长度。 |
| 安全提示 | 首次联网搜索、首次发送正文到模型时展示说明。 |

## 6. 数据迁移

所有实体应包含 schemaVersion。迁移策略：

```text
MigrationPlan {
  fromVersion,
  toVersion,
  description,
  dryRun(),
  apply(),
  rollbackHint
}
```

迁移前应自动创建项目备份。迁移失败时，应保留旧数据并提示用户导出备份。

## 7. 基础错误模型

建议统一错误结构：

```text
AppError {
  code,
  message,
  userMessage,
  technicalDetail?,
  recoverable,
  suggestedAction?,
  source: storage | llm | search | editor | export | unknown
}
```

用户界面展示 userMessage；日志记录 technicalDetail。

## 8. 阶段验收

| 验收项 | 通过标准 |
| --- | --- |
| 工程可运行 | Flutter App 可启动，桌面/Web 至少一个平台正常。 |
| 存储可用 | 可创建、读取、更新、删除 Project 与 Chapter。 |
| 设置可保存 | 服务商配置关闭重开后仍存在，API Key 不以明文展示。 |
| 错误可展示 | 模拟存储失败和模型配置缺失时，UI 有明确提示。 |
| 测试覆盖 | Repository、设置、序列化、迁移至少有基础测试。 |
